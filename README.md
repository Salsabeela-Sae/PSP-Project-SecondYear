## PSP-Project-SecondYear
โปรเจกต์ **SE-STORE** เป็นการพัฒนาระบบจัดการร้านค้าผ่าน **Java** โดยใช้หลักการ **Personal Software Process (PSP)** ในขั้นตอน PSP 0 และ PSP 0.1 ซึ่งเน้นการติดตามเวลา การบันทึกข้อบกพร่อง และการปรับปรุงกระบวนการพัฒนาแบบต่อเนื่อง
เป็นการเขียนโปรแกรมตามโจทย์ในแต่ละสัปดาห์ของคาบเรียน ตามเวลาที่กำหนด

---

## ภาพรวมโปรเจกต์

SE-STORE ประกอบด้วย **10 โปรแกรมเวอร์ชัน** ที่พัฒนาต่อยอดกันจากฟีเจอร์พื้นฐานไปจนถึงระบบจัดการร้านค้าที่สมบูรณ์แบบ ทุกเวอร์ชันทำงานตามแนวทาง PSP เพื่อวิเคราะห์เวลาและคุณภาพของงาน รวมถึงปรับปรุงการพัฒนาตาม Feedback ของตัวเอง

## การใช้งาน PSP ในโปรเจกต์

### PSP 0 (Baseline)
- บันทึกขั้นตอนการวางแผน การออกแบบ Flowchart
- บันทึกเวลาในการพัฒนา (Time Tracking)
- บันทึกข้อบกพร่องพื้นฐาน
- บันทึกข้อมูลการวางแผนและการการจัดทำ Estimate&Actual เพื่อจัดทำ Plan Summary Report
- วางแผนและพัฒนาระบบด้วยความรับผิดชอบส่วนตัว

### PSP 0.1 (Enhanced)
- เพิ่มการป้องกันข้อบกพร่องและติดตามอย่างละเอียด
- ปรับปรุงคุณภาพของโค้ดและการออกแบบอย่างต่อเนื่อง

---

## ไฮไลท์การพัฒนา
### การออกแบบและ Workflow
- มี **Flowchart** แสดงขั้นตอนการทำงานแต่ละเวอร์ชัน
- เพิ่มประสิทธิภาพและความเข้าใจในการพัฒนาเวอร์ชันต่อมา
  
### การติดตามเวลา
- บันทึกเวลาเปรียบเทียบระหว่างเวลาที่วางแผนกับเวลาที่ใช้จริง
- วิเคราะห์ความคลาดเคลื่อนเพื่อปรับปรุงประสิทธิภาพ

### การบันทึกข้อบกพร่อง
- บันทึกจำนวนข้อบกพร่องและเวลาที่ใช้ในการแก้ไข
- โปรแกรมทั้งหมดไม่มีข้อบกพร่องในขั้น Compile และ Test

---

## สิ่งที่ได้เรียนรู้

- **PSP:** เข้าใจหลักการพื้นฐานและสามารถนำไปประยุกต์ใช้ได้จริง
- **Planning&Design:** พัฒนาด้านการบริหารเวลาและการวางแผน รวมไปถึงพัฒยาการออกแบบฐานข้อมูลความรู้ เพืื่อให้ง่ายต่อการเขียนโปรแกรมขึ้นเรื่อย ๆ
- **Java Development:** พัฒนาทักษะจากโปรแกรมพื้นฐานจนถึงระบบเต็มรูปแบบ  
- **Process Improvement:** ระบุจุดอ่อนและปรับปรุงขั้นตอนการทำงานแบบต่อเนื่อง  

---

## เอกสารประกอบแต่ละเวอร์ชัน

- **Development Plan:** วางเป้าหมายและวิธีการพัฒนาสำหรับแต่ละเวอร์ชัน  
- **Time Recording Logs:** วิเคราะห์เวลาเปรียบเทียบระหว่างวางแผนกับทำจริง  
- **Defect Recording Logs:** ติดตามข้อบกพร่องและวิธีแก้ไข  
- **Flowchart:** แสดง Workflow ของแต่ละเวอร์ชัน
